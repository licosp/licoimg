# プッシュに関する行動規範 (Ai)

## 目的
リモートリポジトリに変更を共有する前に、コードの安全性と品質を保証すること。

## 行動指針

### 1. プッシュ前の検証
- **テスト**: プッシュする前に必ずテストが通過することを確認します。
- **品質**: コードがプロジェクトの品質基準（Lintなど）を満たしているか確認します。
- **履歴**: `git log` を確認し、コミット履歴が適切であることを確認します。

### 2. 実行
- **コマンド**: `git push origin <branch>` を使用します。
- **強制プッシュ**: `git push -f` は、そのブランチを自分だけが使用していることが確実な場合を除き、**原則として使用しません**。
