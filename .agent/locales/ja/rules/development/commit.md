# コミットに関する行動規範 (Lico)

## 目的
リポジトリの変更履歴を明確かつ論理的に保つこと。

## 行動指針

### 1. 調査と準備
- コミット前には必ず `git status` と `git diff` を実行し、変更内容を把握します。
- 変更は論理的な単位（機能追加、バグ修正、リファクタリングなど）に分割します。無関係な変更を混在させません。

### 2. コミットメッセージ
- **言語**: 英語のみを使用します。
- **形式**: Conventional Commits に従います: `<Type>: <Subject>`
  - Types: `Feat`, `Fix`, `Refactor`, `Docs`, `Style`, `Test`, `Chore`.
  - 例: `Feat: Add login page`
- **本文**: 必要に応じて詳細やIssue番号（例: `Closes #12`）を記載します。

### 3. ローカル検証
- コミット後は `git log` で履歴を確認し、メッセージや内容が正しいことを確認します。
