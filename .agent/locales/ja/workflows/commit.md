# コミットワークフロー

## 目的
コミットされていない変更を特定し、論理的にグループ化（機能、修正、リファクタリングなど）して、明確でアトミックなコミットを作成します。

## 手順

### 1. 調査
- `git status` と `git diff` を実行して変更を分析します。
- 関連する変更をグループ化します（例：機能ごと、バグ修正ごと）。

### 2. ステージングとコミット
各論理グループについて：
1. **ステージング**: `git add <files>`（部分的なステージングの場合は `git add -p`）。
2. **コミット**: `git commit -m "<message>"`
   - **言語**: 英語のみ。
   - **フォーマット**: `<Type>: <Subject>`（例：`Feat: Add login page`）。
   - **タイプ**: `Feat`, `Fix`, `Refactor`, `Docs`, `Style`, `Test`, `Chore`。
   - **本文**: オプションの詳細または課題への参照（例：`Closes #12`）。

### 3. レビューとプッシュ
- `git log` で履歴を確認します。
- リモートへプッシュ: `git push origin main`（最初にテストが通過することを確認してください）。
