# コミットワークフロー

## 目的
未コミットの変更を特定し、論理的（機能、修正、リファクタリングなど）にグループ化して、明確で原子的なコミットを作成します。

## 手順

### 1. 調査
- `git status` と `git diff` を実行して変更を分析します。
- 関連する変更をグループ化します（例：機能ごと、バグ修正ごと）。

### 2. ステージングとコミット
各論理グループについて：
1. **ステージング**: `git add <files>`（部分的なステージングには `git add -p`）。
2. **コミット**: `git commit -m "<message>"`
   - **言語**: 英語のみ。
   - **フォーマット**: `<Type>: <Subject>`（例：`Feat: Add login page`）。
   - **タイプ**: `Feat`, `Fix`, `Refactor`, `Docs`, `Style`, `Test`, `Chore`。
   - **本文**: 任意の詳細または課題への参照（例：`Closes #12`）。

### 3. レビューとプッシュ
- `git log` で履歴を確認します。
- リモートへプッシュ：`git push origin main`（最初にテストが通ることを確認してください）。
